combined reducers

The VideAppState class is a combined reducer of six core reducers. The listen to the named events. All reducers listen to RESTORE_STATE when a state is to be restored and RESET_STATE when the application is to be resetted to the starting state.

  • handleEdition
    This reducer handles edition events to select, change or activate an edition.
    • RECEIVE_EDITIONS when the edition list is received from the server
    • HIGHLIGHT_EDITION an edition is clicked
    • ACTIVATE_EDITION
    • DEACTIVATE_EDITION
  • handleNetwork
    This reducer handles network operations, where the client asks the server for some data.
    • REQUEST_EDITIONS request the edition list from the server
    • RECEIVE_EDITIONS when the edition list is received from the server
    • RECEIVE_EDITIONS_FAILED when the request failed
    • HIGHLIGHT_EDITION an edition is clicked
    • START_LOADING_DATA when a network operation is started
    • STOP_LOADING_DATA when a network operation is finished
  • handleViews
    This reducer handles views to change between facsimile-, transcription- and page overlay-view
    • SET_VIEW_RATIO
    • SET_VIEW_LAYOUT one of INTRODUCTION, SINGLE_VIEW, VERTICAL_SPLIT, HORIZONTAL_SPLIT, EDITION_SELECTION
    • SET_FIRST_VIEW set module in first view
    • SET_SECOND_VIEW set module in second view
    • SET_SYNC_VIEWS
    • CONFIRM_VIEW
    • ACTIVATE_EDITION
    • DEACTIVATE_EDITION
  • handlePreferences
    This reducer handles preferences.
    • SHOW_PREFERENCES
    • HIDE_PREFERENCES
    • SHOW_ABOUT
    • HIDE_ABOUT
    • SWITCH_LANGUAGE
  • handleTour
    This reducer manages a tour through the application.
    • CLOSE_TOUR
    • LOAD_TOURSTEP
  • handleContextMenu This reducer manages the contextMenu
    • OPEN_CONTEXTMENU
